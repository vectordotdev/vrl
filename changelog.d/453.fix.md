Fixed a confusing compile error where a fallible call earlier in a block could cause a later, unrelated assignment to be reported as the problem. For example:

```coffee
for_each(.) -> |key, value| {
    e = {}
    set(e, [key], value)         # the unhandled error is actually here
    output = push(output, e)     # but the compiler used to flag this line
}
```

The error is now reported on the actual fallible expression, so adding `!` (`set!(...)`) or the `, err =` form fixes it where you'd expect.
