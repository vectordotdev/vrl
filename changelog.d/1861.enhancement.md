Several stdlib functions now declare element-kind constraints on array parameters, so the compiler can detect type mismatches at compile time. For example, passing `[1, 2, 3]` to `join` is now a hard compile error instead of a runtime failure, and passing an `array<bytes>` is inferred as infallible without any `!` or `??`.

Affected functions: `join`, `contains_all`, `tally`, `encode_key_value`, `ip_cidr_contains`, `parse_groks`.
